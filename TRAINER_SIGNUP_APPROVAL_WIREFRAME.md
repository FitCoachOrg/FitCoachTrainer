## 7. Database Schema for Trainer Applications

```sql
-- Trainer Applications Table
CREATE TABLE trainer_applications (
    id SERIAL PRIMARY KEY,
    application_id VARCHAR(20) UNIQUE NOT NULL, -- TR-2024-001
    trainer_email VARCHAR(255) NOT NULL,
    first_name VARCHAR(100) NOT NULL,
    last_name VARCHAR(100) NOT NULL,
    phone VARCHAR(20),
    date_of_birth DATE,
    business_name VARCHAR(255),
    website VARCHAR(255),
    experience_years INTEGER,
    
    -- Certifications (JSONB for flexibility)
    certifications JSONB,
    certification_files TEXT[], -- Array of file URLs
    
    -- Specialties
    specialties TEXT[],
    client_populations TEXT[],
    
    -- Business info
    service_offerings TEXT[],
    session_rate DECIMAL(10,2),
    package_rates_available BOOLEAN,
    online_training_rate DECIMAL(10,2),
    availability_days INTEGER[], -- [1,2,3,4,5,6,7]
    preferred_hours VARCHAR(50),
    
    -- References
    references JSONB, -- Array of reference objects
    testimonials TEXT[],
    
    -- Application status
    status VARCHAR(20) DEFAULT 'pending', -- pending, under_review, approved, rejected
    submitted_at TIMESTAMP DEFAULT NOW(),
    reviewed_at TIMESTAMP,
    reviewed_by UUID REFERENCES trainer(id),
    decision_notes TEXT,
    
    -- Agreement flags
    terms_accepted BOOLEAN DEFAULT false,
    privacy_accepted BOOLEAN DEFAULT false,
    background_check_consent BOOLEAN DEFAULT false,
    commission_agreement BOOLEAN DEFAULT false,
    
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- Indexes for performance
CREATE INDEX idx_trainer_applications_status ON trainer_applications(status);
CREATE INDEX idx_trainer_applications_email ON trainer_applications(trainer_email);
CREATE INDEX idx_trainer_applications_submitted_at ON trainer_applications(submitted_at);
```

## 9. Key User Experience Considerations

### For Applicants:
- **Progress Indicator**: Clear step-by-step progress
- **Save & Resume**: Ability to save incomplete applications
- **Mobile Responsive**: Full functionality on mobile devices
- **Clear Expectations**: Transparent timeline and requirements

### For Admins:
- **Efficient Review**: Quick access to all relevant information
- **Bulk Actions**: Approve/reject multiple applications
- **Communication Tools**: Easy way to request additional info
- **Decision Tracking**: Audit trail for all decisions

### For System:
- **Data Validation**: Comprehensive input validation
- **File Management**: Secure storage of certifications
- **Email Automation**: Timely notifications
- **Integration Ready**: Easy to connect with existing systems
