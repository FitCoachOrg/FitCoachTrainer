Professional Calendar - Custom Client Functionality Test

Expected Behavior After Fix:

  1. When opening the "Add Event" modal, the client name dropdown should show "Select existing client or add custom name"
  2. User can select "+ Add Custom Name" from the dropdown
  3. When "+ Add Custom Name" is selected, a text input field should appear below the dropdown
  4. The text input should have a blue border and show "Enter custom client name" as placeholder
  5. User should be able to type any custom client name in this field
  6. A blue indicator should show "✏️ Custom client - not in your client list"
  7. User can also type in the email field for the custom client
  8. The form should save successfully with the custom client name and email

Test Steps:

  1. Navigate to the Professional Calendar component
  2. Click "Add Event" button
  3. In the client name section, verify the dropdown shows the correct placeholder
  4. Select "+ Add Custom Name" from the dropdown
  5. Verify the custom input field appears with blue styling
  6. Type a custom client name (e.g., "John Doe")
  7. Type a custom email (e.g., "john.doe@example.com")
  8. Fill in other required fields (event name, type, time, duration, date)
  9. Click "Create Event"
  10. Verify the event is created successfully with the custom client information

Key Changes Made:

  • Fixed getSelectedValue() function: Now returns 'custom' when no client name is set, instead of empty string
  • Added shouldShowCustomInput() function: Determines when to show the custom input field
  • Updated input field visibility logic: Uses shouldShowCustomInput() instead of checking if clientName exists
  • Improved placeholder text: More descriptive placeholder for the dropdown
  • Enhanced visual feedback: Blue border and indicator text for custom client mode
  • Removed disabled state: "+ Add Custom Name" option is always available
